# Desarrollador FullsTack Test.

Este proyecto contiene varios ejercicios diseñados para evaluar habilidades en desarrollo backend. A continuación se detallan los ejercicios realizados

Ejercicio 1: Consumo de APIs
Objetivo
Crear una aplicación Node.js que consuma la API de GitHub y muestre los 10 repositorios más populares del usuario "google".

Requisitos Previos
Node.js (https://nodejs.org/)
npm (https://www.npmjs.com/)

Configuración
Clona este repositorio o descarga los archivos en un directorio.

Abre una terminal en el directorio del proyecto y navega a la carpeta del ejercicio:
cd ejercicio_1_consumo_de_apis
Instala las dependencias necesarias ejecutando el siguiente comando:
npm install

Ejecución
Ejecuta el script con el siguiente comando:
node index.js
La aplicación mostrará en la consola los 10 repositorios más populares del usuario "google" ordenados por el número de estrellas en orden descendente


Ejercicio 2: Nomenclatura
Objetivo
Reemplazar los nombres de las variables con nombres más descriptivos que reflejen mejor su función en el código proporcionado.

Descripción
El código original utilizaba nombres de variables poco descriptivos. Se ha refactorizado para utilizar nombres más claros y significativos, mejorando la legibilidad y mantenibilidad del código. No es necesario ejecutar este ejercicio, simplemente se ha mejorado la claridad del código.


Ejercicio 3: Pensamiento Lógico
Objetivo
Escribir una función que tome un número entero como entrada y devuelva un array con todos los números enteros impares desde 1 hasta el número de entrada.

Requisitos Previos
Un navegador web moderno.
Configuración
Clona este repositorio o descarga los archivos en un directorio.

Navega a la carpeta del ejercicio:
cd ejercicio_3_pensamiento_logico

Asegúrate de que los archivos index.html y pensamientoLogico.js estén en la misma carpeta.

Ejecución
Abre el archivo index.html en un navegador web.
Ingresa un número en el campo de entrada y haz clic en el botón "Show Odd Numbers".
Los números impares desde 1 hasta el número ingresado se mostrarán en la página.


Ejercicio 4: Modelado de Bases de Datos
Objetivo
Diseñar un modelo de base de datos para un sistema de gestión de vídeos, incluyendo tablas para vídeos, autores, colaboradores, comentarios, reviews y usuarios.

Requisitos Previos
PostgreSQL (https://www.postgresql.org/)
Un cliente SQL como DBeaver (https://dbeaver.io/)
Configuración de la Base de Datos
Crea una base de datos PostgreSQL y las tablas necesarias ejecutando los comandos SQL proporcionados en database_model.sql .
Ejecución
Abre DBeaver o tu cliente SQL preferido.
Conéctate a tu base de datos PostgreSQL.
Ejecuta los comandos SQL para crear las tablas y relaciones necesarias.


Ejercicio 5: Arquitectura del Backend
Objetivo
Describir cómo estructurar el backend de una aplicación de comercio electrónico, incluyendo tecnologías utilizadas, organización de archivos y uso de patrones de diseño.

Descripción
La arquitectura propuesta incluye:

Tecnologías: Node.js, Express.js, MongoDB, JWT para autenticación.
Estructura de Archivos: Organización en directorios para configuración, controladores, modelos, rutas y middleware.
Patrones de Diseño: MVC (Modelo-Vista-Controlador), inyección de dependencias, patrón Repository, patrón Middleware.
Seguridad: Validación de entrada de usuario, protección contra CSRF


Ejercicio 6: Nomenclatura
Objetivo
Crear un documento de políticas de nomenclatura para el equipo de desarrollo de una compañía.

Descripción
El documento incluye políticas de nomenclatura para bases de datos, variables, funciones, clases, git, etc., asegurando consistencia y claridad en el código.
